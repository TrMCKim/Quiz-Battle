# Quiz Battle — Product Requirements Document (PRD)
> LLM 개발 지시용 문서 | v1.2 | 2026-06-28

---

## 0. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | Quiz Battle |
| 목적 | 교실 내 실시간 문제풀기 배틀 — 학생 간 경쟁·성취감 제공 |
| 사용자 | 관리자(교사) 1명 + 학생 최대 30명 동시 접속 |
| 플랫폼 | 웹(PC 전용, 모바일 미지원) |
| 기술스택 | Node.js / Firebase Realtime DB / Vercel / GitHub |
| 에디터 | Antigravity |
| 외부 API | 사용 안 함 |
| 관리자 ID | `admin` / 초기 PW: `2026` (배포 전 반드시 변경) |

## 1. 화면 구성 (라우팅 트리)

```
/login (로그인 화면)
├── /student/wait          학생 대기 화면
│   └── /student/quiz      학생 문제 풀이 화면
│       └── /student/result 학생 결과 화면
├── /admin/wait            관리자 대기 화면

│   └── /admin/db          문제 데이터베이스 관리 화면
│       └── /admin/add     문제 추가 화면
```

## 2. 공통 UI 요소 (전체 페이지 적용)

### 2-1. 공통 Footer (모든 페이지 최하단 필수)

모든 페이지(`/login`, `/student/*`, `/admin/*`) 최하단에 아래 Footer를 고정 표시한다.

```
© 2026 Quiz Battle. All rights reserved.
이용약관 | 개인정보처리방침
개인정보책임자: 김문창 교사 (고척고등학교) | 문의: 02-1234-5678
```

- **「이용약관」** 텍스트: 클릭 시 모달 팝업으로 [섹션 11. 이용약관 전문] 표시
- **「개인정보처리방침」** 텍스트: 클릭 시 모달 팝업으로 [섹션 12. 개인정보처리방침 전문] 표시
- 모달 팝업 스펙:
  - 화면 중앙, 최대 너비 700px, 최대 높이 80vh, 내부 스크롤 가능
  - 우측 상단 닫기(×) 버튼 제공
  - 배경 반투명 오버레이 처리
  - 내용은 마크다운 렌더링(헤드라인, 줄바꿈, 강조 지원)

### 2-2. 공통 디자인 원칙

- 교육용 서비스에 맞는 깔끔하고 단정한 디자인
- 주요 강조 색상: 단일 계열(밝은 파란 계열) 사용, 무지개 팔레트 금지
- 반응형 불필요(PC 고정 레이아웃)

## 3. 화면별 기능 명세

### 3-1. 로그인 화면 (`/login`)

**진입 시 윤리 가이드 동의 플로우:**
1. 화면 전면에 `윤리가이드.png` 이미지 표시
2. 하단에 문구: `"나는 윤리 가이드 내용을 빠짐없이 읽었고 이 내용을 준수하겠습니다."`
3. 동의 버튼 클릭 → 로그인 화면 본체로 이동 (미동의 시 이동 불가)

**로그인 화면 본체:**
- 별명 입력창 표시
- 입력값이 `admin`이 아닌 경우 → 학생 플로우
  - 게임 미시작: 별명 입력 → `/student/wait` 이동
  - 게임 진행 중: 관리자 허가 팝업 대기 후 승인 시 입장, 거절 시 대기
- 입력값이 `admin`인 경우 → 비밀번호 입력칸 노출
  - 비밀번호 일치: `/admin/wait` 이동
  - 불일치: 에러 메시지 표시

---

### 3-2. 학생 대기 화면 (`/student/wait`)

- 화면 중앙: `"게임이 시작할 때까지 기다려 주세요"` 문구
- 대기 안내 일러스트 이미지(LLM이 SVG 등으로 생성) 표시
- 관리자가 게임 시작 → 5, 4, 3, 2, 1 카운트다운 후 `/student/quiz` 이동
- 관리자가 별명 변경 시 → 변경된 별명 실시간 반영 표시 (학생 수정 불가)
- 관리자가 추방 시 → `"퇴장 처리되었습니다"` 안내 표시

---

### 3-3. 학생 문제 풀이 화면 (`/student/quiz`)

**레이아웃 (1:3 비율):**

```
┌─────────────────┬─────────────────────────────────────┐
│   대시보드(1)    │           문제 영역(3)              │
└─────────────────┴─────────────────────────────────────┘
```

**대시보드 표시 항목:**
- 정답 수 / 오답 수 / 남은 문제 수
- 내 문제 풀이 현황: 정답(O) / 오답(X) / 미풀이(_) 순서대로 표시
- 문제 풀이 경과 시간
- 남은 시간 (게임 전체 타이머)
- 등수 표시 문구 (강조 표시):
  - 일반: `"현재 [별명]님은 X등이고 바로 앞은 [별명1]님, 바로 뒤는 [별명2]님입니다"`
  - 1등: `"훌륭합니다. 현재 [별명]님은 1등이고 바로 뒤는 [별명1]님입니다."`
  - 꼴등: `"힘내세요. 현재 [별명]님은 꼴등이고 바로 앞은 [별명1]님입니다."`
- 이벤트 로그: 최신 이벤트가 맨 위, 시간순 역순 표시 (새 이벤트 추가 시 기존 항목 한 줄씩 아래로)

**문제 영역:**
- 표시 순서: 문제 번호 → 본문 → 그림(없으면 생략) → 보기(선택형만)
- 글자 크기: 기본 12pt, 내용 길이에 따라 한 화면에 표시되도록 자동 조절
- 그림: 원본 크기 원칙, 화면 초과 시 크기 자동 조절
- **선택형**: 보기를 마우스로 클릭 (키보드 입력 불가)
- **단답형**: 답안 입력칸 표시, 커서 자동 포커스, 키보드 입력만 가능
- 오른쪽 하단 버튼:
  - `[Skip]`: 현재 문제 포기 → 다음 문제 이동
  - `[제출]`: 답 입력 후에만 활성화
- 제출 결과 피드백 (1초 표시 후 다음 문제 이동):
  - 정답: `"축하합니다. 정답입니다!"` (크게 표시)
  - 오답: `"아쉽습니다. 오답입니다."` (크게 표시)

---

### 3-4. 학생 결과 화면 (`/student/result`)

**대기 상태 (다른 학생 풀이 중):**
- 게임 종료 일러스트 이미지(LLM이 SVG 등으로 생성) 표시
- `"다른 학생이 문제를 풀고 있는 중입니다. 잠시만 기다려 주세요"` 문구

**결과 확인 상태 (전원 완료 후):**

```
┌────────────────────────────────────────────────────┐
│              결과 안내 문구                         │
├───────────────────┬────────────────────────────────┤
│   이벤트 확인     │        문제 확인               │
└───────────────────┴────────────────────────────────┘
```

- **결과 안내 문구**: `"수고하셨습니다. [별명]님은 정답 X문제, 오답 X문제입니다."` + `"[별명]님은 X등을 하셨습니다."`
- **이벤트 확인**: 게임 중 발생 이벤트 + 발생 시각 목록, 많으면 스크롤
- **문제 확인**:
  - 첫 번째 문제부터 순서대로 표시
  - 표시: 문제 번호 / 본문 / 그림(있으면) / 보기(선택형) / 정답 / 학생 정오 여부 / 출처(있으면)
  - 오른쪽 하단: `[이전 문제]` `[다음 문제]` 버튼 (첫 문제에서 이전 버튼 숨김, 마지막 문제에서 다음 버튼 숨김)

---

### 3-5. 관리자 대기 화면 (`/admin/wait`)

**레이아웃:**
- 왼쪽: 접속 학생 별명 목록
- 오른쪽: 게임 시작 버튼(크게)
- 우측 상단: 문제 관리 버튼 → `/admin/db` 이동

**학생 목록 기능:**
- 각 학생 행: `[별명] [수정] [추방]`
  - 수정 버튼: 해당 학생 별명 수정 (변경 내용 학생 화면에 실시간 반영)
  - 추방 버튼: 해당 학생 접속 종료 처리

**게임 설정:**
- 게임 시간 설정: 기본 10분, 최대 30분 (스크롤/드롭다운 선택)

**게임 시작:**
- 게임 시작 버튼 클릭 → `/admin/progress` 이동 + 학생 화면 카운트다운 시작

---

### 3-6. 문제 데이터베이스 관리 화면 (`/admin/db`)

**레이아웃:**
```
┌──────────────────────────────────────────────────────┐
│  [문제 추가]              문항 수: [스크롤] [자동선택] │
├──────────────────────────────────────────────────────┤
│                       문제 목록                       │
│   (체크박스 - 문제 내용 - 문제 정보 카드 반복)        │
└──────────────────────────────────────────────────────┘
```

**문항 수 설정:**
- 범위: 5 ~ DB 내 문제 수(최대 100)
- 기본값: 10 (DB 문제가 10개 미만이면 DB 전체)

**문제 목록:**
- 둥근 모서리 카드 형태, 전체 스크롤
- 카드 구성: 체크박스 / 본문 / 그림(있으면) / 보기(선택형) / 정답 / 출처 / 학년 / 단원
- 체크된 문제 = 출제 대상
- 정렬·필터: 출처 / 학년 / 단원 기준 제공

**하단 버튼:**
- `[문제 선택 완료]` 클릭 → 선택 문제 확정 후 `/admin/wait` 이동

---

### 3-7. 문제 추가 화면 (`/admin/add`)

| 항목 | 입력 방식 | 필수 여부 |
|------|-----------|-----------|
| 문제유형 | 드롭다운 (선택형 / 단답형) | 필수 |
| 본문 | 텍스트 입력 (LaTeX 수식 입력기 포함) | 필수 |
| 보기 | ①②(필수) ~ ⑤(선택) 텍스트 입력. 선택형만 표시. 보기 추가 버튼(5개 초과 시 에러) | 선택형일 때 필수 |
| 그림 | 파일 업로드 (jpg/png/gif, 1MB 이하. 초과 시 에러) | 선택 |
| 정답 | 선택형: 보기 번호 드롭다운 / 단답형: 텍스트 입력 | 필수 |
| 출처 | 텍스트 입력 (LaTeX 수식 입력기 포함) | 선택 |
| 학년 | 텍스트 입력 | 선택 |
| 단원 | 텍스트 입력 (LaTeX 수식 입력기 포함) | 선택 |

**LaTeX 수식 입력기:**
- 텍스트 입력 칸 전체에 수식 입력기 적용
- 입력 중 미리보기(Preview) 실시간 렌더링

**저장:**
- 오른쪽 하단 `[저장]` 버튼
- 필수 항목 미입력 시 에러 메시지 표시 후 저장 중단

**뒤로 가기:**
- 왼쪽 상단 `[← 문제 데이터베이스 관리]` 버튼
- 클릭 시 `/admin/db` 화면으로 이동
- 입력 중인 내용이 있을 경우 `"작성 중인 내용이 있습니다. 저장하지 않고 나가시겠습니까?"` 확인 팝업 표시 → 확인 시 이동, 취소 시 현재 화면 유지

---

### 3-8. 문제 풀이 진행 관리 화면 (`/admin/progress`)

- 모든 학생 진행 현황 실시간 표시:
  - 현재 풀이 중인 문제 번호 / 정답 수 / 오답 수 / 현재 등수
- 전체 남은 시간 표시
- 게임 중 신규 입장 요청 팝업: 학생 별명 표시 → `[허가]` `[거절]` 선택
- `[문제 풀이 종료]` 버튼: 클릭 시 게임 즉시 종료 → `/admin/result` 이동
- 화면 구성은 30명 동시 모니터링에 효율적인 그리드 레이아웃으로 제안할 것

---

### 3-9. 문제 풀이 종료 관리 화면 (`/admin/result`)

- 게임 종료 후 결과를 효율적으로 확인할 수 있는 화면 구성 (LLM이 최적 레이아웃 제안)
- 표시 항목: 학생별 정답 수 / 오답 수 / 풀이 시간 / 문항별 정오 여부 / 오답 내용

> **[삭제됨]** ~~CSV 파일 다운로드 기능~~ — 요구사항에서 제거됨

## 4. 등수 산정 원칙

우선순위 순서:
1. 정답 수 많은 순
2. 정답률 높은 순 (Skip 포함 전체 문제 기준)
3. 평균 문제 풀이 시간 짧은 순

## 5. 이벤트 시스템

| 이벤트명 | 발생 조건 | 표시 문구 | 표시 대상 |
|----------|-----------|-----------|-----------|
| 연속 정답 | 연속 3회 이상 정답 | `"연속하여 X번 정답입니다"` (X=연속 횟수) | 해당 학생만 |
| 최초 정답 | 특정 문제의 첫 번째 정답 | `"[별명]님이 X번 문제를 최초로 해결하였습니다"` | 해당 학생만 |
| 순위 역전 | 두 학생의 등수 교체 | `"[별명1]님이 [별명2]님을 추월하였습니다"` | 역전 당사자 2명 |

- 이벤트는 발생 즉시 Firebase Realtime DB를 통해 해당 학생 화면에 실시간 반영
- 이벤트 로그: 발생 시각 + 이벤트 문구, 최신 순(역순) 표시

## 6. 데이터베이스 설계 (Firebase Realtime DB)

### 6-1. 문제 컬렉션 (`/questions/{id}`)

```json
{
  "type": "choice | short",
  "body": "문제 본문 (LaTeX 포함 가능)",
  "options": ["①보기", "②보기", "③보기"],
  "imageUrl": "Firebase Storage URL | null",
  "answer": "보기 번호(선택형) | 텍스트(단답형)",
  "source": "출처 | null",
  "grade": "학년 | null",
  "unit": "단원 | null"
}
```

- 최대 100개 제한
- 문제 풀이 결과는 DB에 저장하지 않음

### 6-2. 관리자 인증 (`/admin/password`)

```json
{
  "hash": "bcrypt 해시값"
}
```

- 평문 저장 금지, bcrypt 해시 저장

### 6-3. 게임 세션 (`/session`) — 인메모리(비영구)

```json
{
  "status": "waiting | countdown | playing | finished",
  "timeLimit": 600,
  "startedAt": 1234567890,
  "players": {
    "playerKey": {
      "nickname": "별명",
      "answers": {},
      "events": []
    }
  }
}
```

- 세션 데이터는 게임 종료 후 파기 (DB 비저장)
- 별명은 세션 중에만 존재, 별도 저장 안 함

---

## 7. 실시간 통신

- Firebase Realtime DB `onValue` 리스너로 학생·관리자 간 실시간 상태 동기화
- 대상 이벤트: 게임 시작/종료, 문제 진행 현황, 등수, 이벤트 로그, 별명 변경, 추방

## 8. 보안 요구사항

## 11. 이용약관 전문 (모달 팝업 콘텐츠)

> 아래 내용을 「이용약관」 링크 클릭 시 모달로 표시할 것

본 이용약관(이하 '약관')은 퀴즈 배틀(Quiz-Battle)(이하 '본 서비스')이 제공하는 교육용 웹 애플리케이션 서비스의 이용에 관한 사항을 규정합니다.

### 제1조 (목적)

이 약관은 본 서비스가 제공하는 무료 교육용 웹 애플리케이션 서비스(이하 '서비스')를 이용함에 있어 서비스 제공자와 이용자의 권리·의무 및 책임사항을 규정함을 목적으로 합니다.

### 제2조 (정의)

- '서비스'란 본 플랫폼에서 제공하는 교육용 웹 애플리케이션을 말합니다.
- '이용자'란 본 서비스에 접속하여 이 약관에 따라 서비스를 이용하는 회원 및 비회원을 말합니다.
- '회원'이란 본 서비스에 회원등록을 한 자로서, 서비스를 이용할 수 있는 자를 말합니다.

### 제3조 (약관의 명시와 개정)

① 본 서비스는 이 약관의 내용을 이용자가 쉽게 알 수 있도록 서비스 초기 화면에 게시합니다.

② 본 서비스는 관련 법령을 위배하지 않는 범위에서 이 약관을 개정할 수 있습니다.

③ 약관을 개정할 경우에는 적용일자 및 개정사유를 명시하여 현행약관과 함께 서비스 내에 그 적용일자 7일 이전부터 공지합니다.

### 제4조 (서비스의 제공)

- 본 서비스는 교육 목적의 무료 웹 애플리케이션을 제공합니다.
- 서비스의 이용은 무료이며, 별도의 유료 결제가 필요하지 않습니다.
- 본 서비스는 교육 활동 지원을 목적으로 하며, 상업적 목적으로 운영되지 않습니다.

### 제5조 (서비스의 중단)

① 본 서비스는 시스템 점검, 교체 및 고장, 통신 두절 등의 사유가 발생한 경우에는 서비스의 제공을 일시적으로 중단할 수 있습니다.

② 본 서비스는 무료로 제공되는 교육용 서비스이므로, 서비스 중단으로 인한 별도의 보상은 제공되지 않습니다.

### 제6조 (회원가입)

① 본 서비스는 별도의 회원가입을 받지 않습니다.

② 만 14세 미만의 아동은 본 서비스를 이용할 수 없습니다.

### 제8조 (이용자의 의무)

이용자는 다음 행위를 하여서는 안 됩니다.

- 허위 내용의 등록
- 타인의 정보 도용
- 서비스에 게시된 정보의 무단 변경
- 서비스의 운영을 방해하는 행위
- 타인의 명예를 손상시키거나 불이익을 주는 행위
- 공서양속에 반하는 정보를 게시하는 행위

### 제9조 (저작권)

① 본 서비스가 작성한 저작물에 대한 저작권은 서비스 제공자에게 귀속합니다.

② 이용자는 서비스를 이용하여 얻은 정보를 서비스 제공자의 사전 승낙 없이 복제, 송신, 출판, 배포하여서는 안 됩니다.

### 제10조 (면책조항)

① 본 서비스는 무료로 제공되는 교육용 서비스로서, 서비스 이용 중 발생하는 기술적 문제나 오류에 대해 제한적 책임을 집니다.

② 본 서비스가 연결하는 외부 웹 애플리케이션의 내용에 대해서는 해당 애플리케이션 제공자가 책임을 집니다.

### 제11조 (분쟁해결)

본 서비스와 이용자 간에 발생한 분쟁에 관하여는 대한민국 법을 적용하며, 소송이 제기되는 경우 서비스 제공자의 소재지를 관할하는 법원을 관할법원으로 합니다.

## 12. 개인정보처리방침 전문 (모달 팝업 콘텐츠)

> 아래 내용을 「개인정보처리방침」 링크 클릭 시 모달로 표시할 것

[Quiz Battle](이하 본 서비스)은(는) 개인정보 보호법 제30조에 따라 정보주체의 개인정보를 보호하고 이와 관련한 고충을 신속하고 원활하게 처리할 수 있도록 하기 위하여 다음과 같이 개인정보 처리방침을 수립·공개합니다.

### 제1조 (개인정보의 처리 목적)

본 서비스는 다음의 목적을 위하여 개인정보를 처리합니다. 처리하고 있는 개인정보는 다음의 목적 이외의 용도로는 이용되지 않으며, 이용 목적이 변경되는 경우에는 개인정보 보호법 제18조에 따라 별도의 동의를 받는 등 필요한 조치를 이행할 예정입니다.

- 문제 풀이 입장 시 별명: 문제 풀이 구성원 식별, 교사 피드백 제공

### 제2조 (개인정보의 처리 및 보유기간)

① 본 서비스는 법령에 따른 개인정보 보유·이용기간 또는 정보주체로부터 개인정보를 수집 시에 동의받은 개인정보 보유·이용기간 내에서 개인정보를 처리·보유합니다.

② 각각의 개인정보 처리 및 보유 기간은 다음과 같습니다.

- 보유 기간: 문제 풀이 진행 기간(문제 풀이 종료 후 저장하지 않음)
- 파기 시점: 문제 풀이 종료 후 파기(저장하지 않음)

### 제3조 (처리하는 개인정보 항목)

본 서비스는 학습 지원을 위해 필요한 최소한의 개인정보만을 수집합니다.

- 수집 항목: 별명
- 수집하지 않는 항목: 개인 식별을 위한 별명 이외의 모든 항목

### 제4조 (만 14세 미만 아동의 개인정보 처리에 관한 사항)

① 본 서비스는 만 14세 미만 아동의 사용을 제한합니다.
